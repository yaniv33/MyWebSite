2. Project, short description

MODULES

  1. product-service :
    shows module working in front of mongodb database.

  2. order & inventory services :
    shows a service dependent on another service , each order checks inventory before placing order

  3. discovery-service :
    uses eureka cloud service to register all services , thus allowing access to all modules.
    key words : eureka.
    eureka , user:eureka; pass:pass

  4. api-gateway :
    uses cloud gateway to map the path to all services , thus all request could approach one gateway
    uses spring security to secure connections to the services
    uses keycloak to authorize users based on auth2 authorization process.
    keycloak , user:admin; pass:admin
    key words : cloud gateway. spring security , keycloak

  5. notification-service :
    uses kafka queue managing broker , in order to manage queue for sending messages such as sms / email
    key words : kafka

  6. configuration-service :
    allows storing of all properties / yaml configuration files in one location
    instead of each project holding its own configuration, the configuration service would hold and distribute via http socket

ADDED Utilities

  1. lombok :
    a utility for adding bean utils such as builders ,getters and setters , logs etc..

  2. WebClient :
    part of spring-boot-starter-webflux , after setting configuration, can use webclient builder to sent http requests between services ..

  3. docker(composs) :
    Ext installation, tool for building container that will hold all our cloud utilities..

  4. actuator :
    allows monitoring tools to run on our project.

  5. zipkin(sleuth) :
    a utility for monitoring http request thought the verious services
    zipkin

  6. circuit(resilience4j, zookeeper) :
    a utility for managing failed http request , allow circuit states, define attempts num & duration , define process for failure ..

  7. jib :
    pluggin , allows connecting to our docker container

  8. prometheus/ grafana
    allows getting visual information about our projects

3. ADDING SECURITY TO GATEWAY

add to properties

spring.security.oauth2.resourceserver.jwt.issuer-uri= http://localhost:8181/realms/Spring-boot-microservices-realm

add to gradle

implementation 'org.springframework.boot:spring-boot-starter-oauth2-resource-server'
implementation 'org.springframework.boot:spring-boot-starter-security'

add security config
*The security config will accept jwt requests authenticated by keycloak OAuth mechanize *

@Configuration
@EnableWebFluxSecurity // we need webflux and not web because it is required by spring cloud gateway
public class SecurityConfig {

    @Bean
    public SecurityWebFilterChain springSecurityFilterChain(ServerHttpSecurity serverHttpSecurity) {
        serverHttpSecurity
                .csrf().disable()
                .authorizeExchange(exchange ->
                        exchange.pathMatchers("/eureka/**")
                                .permitAll()        // if eureka , allow all , static data for the eureka page
                                .anyExchange()
                                .authenticated())   // all the rest needs to be authenticated
                .oauth2ResourceServer(ServerHttpSecurity.OAuth2ResourceServerSpec::jwt);
        return serverHttpSecurity.build();
    }
}

Allow access to eureka webpage via security (DISCOVERY SERVICE)

In order to allow access to aureka service, we must implement the security on the discovery service as well

Example of implementing websecurity with in memory user and password ..

@Configuration
@EnableWebSecurity // need webflux rather than webmvc becuase the spring cloud services work with web flux
public class SecurityConfig{

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {

        http.csrf().disable()
                .authorizeHttpRequests()
                .requestMatchers("/eureka/**").permitAll()
                .anyRequest().authenticated()
                .and()
                .formLogin()
                .and()
                .httpBasic();
        return http.build();
    }

    @Autowired
    public void configureGlobal(AuthenticationManagerBuilder auth) throws Exception {
        auth    .inMemoryAuthentication()
            .passwordEncoder(NoOpPasswordEncoder.getInstance()) // do to : check how to add encoding
                .withUser("eureka").password("password").roles("USER");
    }
}

now the page would be accesible via localhost:8080/eureka/web with user and password

and finally , update the url of eureka service for the gateway service , so that it would include the user and password

eureka.client.serviceUrl.defaultZone=http://eureka:password@localhost:8761/eureka

working with circut

The circut mechanize allows us to deal with failed rest calls, by allowing state change , setting timeout and setting retry

circuit states :

Closed -The circuit breaker is closed and the call goes through
Half Open - The circuit breaker is open and the call fails immediately without any attempt to execute the real operation.
Open - allows limited number of calls to see if the operation has recovered.

add to gradle (actuator is for allowing web view monitoring of the circuit)
implementation 'org.springframework.cloud:spring-cloud-starter-circuitbreaker-reactor-resilience4j'
implementation 'org.springframework.boot:spring-boot-starter-actuator'
set Resilinece4j.circuit properties in the properties file

in the [name] section, place a common name such as "inventory"

#Resilinece4j Properties
resilience4j.circuitbreaker.instances.[name].registerHealthIndicator=true           -- enable circuit
resilience4j.circuitbreaker.instances.[name].event-consumer-buffer-size=10          -- buffer assign for circuit
resilience4j.circuitbreaker.instances.[name].slidingWindowType=COUNT_BASED          -- circuit could be count/time based
resilience4j.circuitbreaker.instances.[name].slidingWindowSize=5                    -- num of event for count
resilience4j.circuitbreaker.instances.[name].failureRateThreshold=50                -- failre rate (%)
resilience4j.circuitbreaker.instances.[name].waitDurationInOpenState=5s             -- wait in open state
resilience4j.circuitbreaker.instances.[name].permittedNumberOfCallsInHalfOpenState=3   -- calls in half open state
resilience4j.circuitbreaker.instances.[name].automaticTransitionFromOpenToHalfOpenEnabled=true -- allow automatic transition to half open state

#Resilience4J Timeout Properties
resilience4j.timelimiter.instances.[name].timeout-duration=3s  -- duration for @TimeLimiter(name = "[name]")

#Resilience4J Retry Properties  , for : (@Retry(name = "[name]"))
resilience4j.retry.instances.[name].max-attempts=3      -- number of retries
resilience4j.retry.instances.[name].wait-duration=5s    -- retry after
set the controller for ciruit use
    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    @CircuitBreaker(name="[name]" , fallbackMethod = "fallbackMethod")  // in case of [name] circuit fail , call method fallbackMethod
    @TimeLimiter(name = "[name]")                                   // use [name] timeout properties
    @Retry(name = "[name]")                                         // use [name] retry properties
    public CompletableFuture<String> placeOrder(@RequestBody OrderRequest orderRequest) {
        return CompletableFuture.supplyAsync(() -> orderService.placeOrder(orderRequest));
    }

    public CompletableFuture<String> fallbackMethod(OrderRequest orderRequest, RuntimeException runtimeException) {
        return CompletableFuture.supplyAsync(() -> "Oops! Something went wrong, please order after some time!");
    }

KAFKA

KAFKA is a utility for managing a broker for sending messages as email or sms

information about kafka can be found here:
https://developer.confluent.io/quickstart/kafka-docker/

add to parent project the file docker-compose.yml

set up docker kafka with : docker compose up -d
see kafka logs with : docker logs -f broker

add kafka implementaion in both client and server services

implementation 'org.springframework.kafka:spring-kafka'

set a common carrier object in both client and server services for carrying the message
use duplicate in order to avoid dependencies between services

@Data
@AllArgsConstructor
@NoArgsConstructor
public class OrderPlacedEvent {
    private String orderNumber;
}

configure application properties on the client side as serializer
[Name of topic] - the broker name that would be used throughout the queue

spring.kafka.bootstrap-servers=localhost:9092
spring.kafka.template.default-topic=[Name of topic]
spring.kafka.producer.key-serializer=org.apache.kafka.common.serialization.StringSerializer
spring.kafka.producer.value-serializer=org.springframework.kafka.support.serializer.JsonSerializer
spring.kafka.producer.properties.spring.json.type.mapping=[path to carrier object]

configure application properties on the server side as deserializer
[topic group id] - the server needs to also define the group id
notice that the mapping id different in both properies as each points to its own identical object

spring.kafka.bootstrap-servers=localhost:9092
spring.kafka.template.default-topic=[Name of topic]
spring.kafka.consumer.group-id= [topic group id]
spring.kafka.consumer.key-deserializer=org.apache.kafka.common.serialization.StringDeserializer
spring.kafka.consumer.value-deserializer=org.springframework.kafka.support.serializer.JsonDeserializer
spring.kafka.consumer.properties.spring.json.type.mapping=event:com.example.notificationservice.OrderPlacedEvent

define the topic endpoint in the server service

    @KafkaListener(topics = "[Name of topic]")
    public void handleNotification(OrderPlacedEvent orderPlacedEvent) {

        log.info("Received Notification for Order - {}", orderPlacedEvent.getOrderNumber());
    }
